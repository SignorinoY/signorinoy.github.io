## 原则和方法

### 性能与可拓展性

#### 性能

提升网站性能即减少特定页面加载的时间或完成对特定用户可见的操作时间.
性能应该完全围绕用户体验展开（比如,有用的内容“立即”显示、不重要的内容（如广告等）延后显示）.

#### 可拓展性

易拓展网站即容易通过添加更多的硬件和网络带宽（却不对软件做重大变更）支持更多的用户,而且对性能的影响微乎其微.

#### 高性能及易拓展的目标

组件|目标
----|------
页面|在负载下,每个页面都是可拓展的,并且响应速度快
层|在负载下,每个层都是可拓展的,并且运行速度快
敏捷性|可以快速地响应业务需求的变化,而且在变化时能保持性能和可拓展性
可维护性|快速地发现和修正性能相关的BUG
运维|快速地部署和扩大网站,容量规划直观且可靠
硬件|服务器在负载下被充分利用,并且尽可能少使用硬件设备

### 完整的页面处理过程

#### 基于ASP.NET和SQL Server的网站中可能影响性能的Web架构组件

![可能影响性能的Web架构组件](https://xn--i0v668g.com/uploads/images/building-a-high-performance-expandable-ASP-Net-website-1)

其中显示的所有组件都会给加载页面带来时间上的延迟,但这种延迟是可以控制在范围内.这里没有包含额外的组件,比如路由器、负载均衡器以及防火墙,因为它们带来的延迟一般是不可控的.

#### 网页加载的过程

1. 首先,浏览器会查看本地缓存是否保存了该网页的副本.
2. 如果本地缓存中没有该网页,则浏览器会通过DNS查询Web服务器的IP地址.浏览器和操作系统都有自己的DNS缓存,用来存储先前查询的结果.如果该地址未知或者缓存项已超时,则会向最近的DNS发起查询（通常就在本地路由器中）.
3. 接下来浏览器会打开一个到Web服务器的网络连接.有时候,该连接可能直接连到代理服务器.该代理服务器要么是可见代理（visible proxy）,要么是透明代理（transparent proxy）.可见代理是用户通过名称配置的,通常在大型公司使用.比如有时是为了提高员工访问Web的性能,有时是为了安全或过滤策略.透明代理无须配置,它会拦截所有从80端口（HTTP）外出的TCP连接,不管本地客户端的配置.如果本地代理没有所期望的内容,则该HTTP请求就会发送到Web服务器.
4. 有些ISP也使用代理来提高用户的访问性能,减少用户使用的带宽.和本地代理一样,如果该内容在ISP的代理缓存里不存在,请求就会继续发送.
5. 下一站就是目标网站的Web服务器.大型网站一般会有多个负载均衡服务器,任何一个都可以接受并处理进来的请求.在IIS和ASP.NET中,每台机器都有自己的硬盘和操作系统驱动层次（http.sys）的独立缓存.
6. 如果请求的页面需要从数据库获取数据,那Web服务器还得打开一个或多个到数据库服务器的连接.然后发出对所需数据的查询.这些数据或许已经在数据库的RAM缓存中,或许还得从硬盘读取.
7. Web服务器得到了所需的数据后,就动态生成请求的页面,并将其发送回用户.如果发送回的结果中有适当的HTTP响应头,这些结果就可以在多处缓存.
8. 响应到达客户端后,浏览器解析结果,将其呈现在屏幕上.

## 客户端性能

显示网页的过程会涉及[分布式计算](https://baike.baidu.com/item/分布式计算/85448?fr=aladdin).即客户端对浏览器负责请求和解析HTML,CSS,JS,图片以及页面上其他对对象,服务器（一台或多台）生成并分发动态和静态的内容.因此,要构建高性能的系统,浏览器端和服务器端都要快,介于两者之间对网络和其他组件也要快.
一般而言,优化网站的输出,让它在客户端上运行得更快通常比服务器端代码运行得更快更能产生大的性能改进.

### 浏览器页面处理

#### 网络连接和初始HTTP请求

> ![网络连接和初始HTTP请求](http://images.51cto.com/files/uploadimg/20110224/152729425.png)
> 浏览器通过发送TCP SYN包让服务器打开一个连接.服务器确认SYN包作出响应,此时连接被打开.
> 然后浏览器发送一个HTTP GET包,包括请求的URL、Cookie等其他细节.稍后,服务器发送ACK包,在这段时间内服务器生成响应（图2-1标记为A的地方）.
> 然后服务器发送响应,而客户端用ACK包回应.ACK请求的频率由TCP”**窗口**“**大小** 确定,它是决定网络速度的重要因素.
> 包的最大长度从500字节到1 500字节不等,这取决于MTU（Maximum Transmission Unit,最大传输单元）.来自服务器的第一个数据包包括HTTP响应头,通常还有一些HTML,这要看头部的大小.因为TCP网络协议的这种工作方式（称之为慢启动（slow start） 的特性）,所以当网络连接提高到全速时,前一个数据包与后一个数据包之间会有很长的延迟.
> SYN和SYN ACK包加上TCP的慢启动特性,致使打开网络连接是相对来说很花时间的过程.因此我们应该避免过多地打开网络连接.

#### 页面解析和新的资源请求

当浏览器等待数据包时,它会解析已经得到的包,并且还会寻找可用的新的HTTP请求,并行启动.对于每个域名,浏览器最多可以同时打开两个连接.

- 在一些大型的网站中,会使用专门的子域名来承载不同的东西（图片,CSS,JS等）
- 利用浏览器这种处理页面的方式,可以将一个或多个请求对象放到HTML的顶部

#### 页面资源的获取以及重新排序

浏览器会在呈现HTML中的`<body>`元素之前请求`<head>`节的所有资源.因为`<head>`节不能包含任何能让浏览器在屏幕上绘制内容的标签,所以在`<head>`部分的所有资源下载完之前,用户是不会看到任何东西的.

- 如果将一些请求放在`<body>`内,那么呈现页面和下载资源就可以并行进行

随着HTML的解析,页面引用的资源（包括图片）都会按照IE碰到的顺序排队获取.浏览器会在请求其他来自同一个域名的资源之前请求位于文件头部的图片

- 根据图片对于用户的优先级而不是他们在HTML中的位置来加载对象

  - 通过Javascript来实现提前或延迟加载

  ``` html
  <img id=“myimg” width=“50” height=“50”>
  <script type=“text/javascript”>
       Document.getElementById(“myimg”).src=“myimage.png”;
  </script>
  ```

  - 通过CSS,按照对于用户的重要程度安排内容在HTML中的顺序
    [圣杯布局](https://xn--i0v668g.com/2017/10/31/HTML%E5%B8%83%E5%B1%80%E4%B9%8B%E5%9C%A3%E6%9D%AF%E5%B8%83%E5%B1%80/) 、 [双飞翼布局](baidu.com)等.

### 浏览器缓存

浏览器的缓存是区分大小写的,但是Windows NTFS文件系统和IIS URL处理是不区分大小写的（与Unix/Linux和Apache是不同）.

- URL和文件使用一致的大小写,避免客户端多次下载同一文件
- 对于动态的内容,在一个ASP.NET HttpModule中检查输入的混合大小写的URL

浏览器缓存将有些内容与具体的URL相关,为获得更好的性能,应该总是使用相同的URL引用相同的内容.

- 如果运行多个站点,可以为共同的静态内容使用共享的域名来改进性能
- 如果多个开发人员开发网站,应注意共享和重用内容

相同的策略同样适用于域名.

- 如果有多个不同的域名引用相同的网站,采用一致的方式使用它们（将不同的域名都转向到一个域名）,可以改进客户端缓存并有利于SEO（搜索引擎会因为相同内容的多个副本而排除或降低你的网站排名）

### HTML优化

#### HTML的一般优化

- ** 使用小写的URL,标签名称,属性名称等**,有助于最大化HTML的压缩率
- 检查并移除冗余标签.比如,如果有两个连在一起的`<span>`标签就应该合并它们
- 移除`<meta refresh>`标签.自动页面刷新乍一看有时很动人,但是考虑这样的情况,用户离开电脑,或者正在看浏览器的另外一个选项卡（tab页）.如果页面因为`<meta refresh>`持续刷新,那么这只是会浪费客户端和服务器的资源
- 移除没有内容的标签
- 从自动化内容生成器生成的内容中移除无关的标签
- 减少alt文本的长度
- 移除注释和额外的空格
- 使用自闭合的标签

#### 使用JavaScript降低HTML的大小

##### 生成重复的HTML

例如,生成有一个全世界所有国家或中国所有省市列表的下拉框：

```html
<div id="myselect"></div>
<script>
    var countryList = "AF,Afghanistan,AL,Albania,DZ,Algeria,US,United State";
    function displayCountries(parentElementID) {
        var countries = countryList.split(",");
        var count = countries.length;
        var parentElement = document.getElementById(parentElementID);
        var select = document.createElement("select");
        for (var i = 0; i < count; i += 2) {
            var option = document.createElement("option");
            option.value = countries[i];
            option.text = countries[i + 1];
            select.appendChild(option);
        }
        parentElement.appendChild(select);
    }
    displayCountries("myselect");
</script>
```

##### 为标签添加重复文本

例如,在页面的很多链接上附加一个很长的查询字符串参数：

```html
<a id=“lk” href=“#” />
<script type=“text/javascript”>
    var l = document.getElementById(‘lk’);
    l.href += “?is_all=1”;
</script>
```

### CSS优化

- 移除没有使用的CSS
- 将CSS style 属性替换为独立的（可被缓存的）CSS文件中的class名,ID或其他选择符
- 合并重复的CSS样式和通用元素,使用[属性继承](baidu.com),简写,层叠以及其他优化措施减小CSS的大小
- 指定0像素时,单位px可以省略
- 当为margin或padding的top、right、bottom、left指定相同的值时,只需要将这些值列一次就行了,无需四次
- 尽可能使用三位十六进制颜色代替六位版本的.三位的代码等同六位的,六位的代码只是将每个字符复制两次.比如：#0a9 与 #00aa99相同
- 使用十六进制颜色代替rgb().下面的标准颜色都有仅占三位的等价表示,比它们的名称要短：black (#000)、fuchsia (#f0f)、white (#fff)、yellow (#ff0)
- 尽量使用CSS速记法.比如,将margin的值列在一起,而不用四个不同的值表示margin-top、margin-right、margin-bottom、margin-left

### JavaScript优化

#### JavaScript的一般优化

- 将`<script>`包含在`</body>`之前或在它们之前放一个或多个图片来促进网络并行化

#### 替代“文本图片”

使用在线字体来替换“文本图片”.
[参考文章](baidu.com)

若该文本的字体很重要,比如商标,那么就可以在页面的onload处理器里使用图片代替文本,如：

```html
<head>
    <style type="text/css">
        .hdr{border:1px solid #000;height:40px;
        background:url(top-grad.gif)}  
        #txtlogo{font-family:lucida handwriting,cursive;  
        font-size:32px;color:#fff;padding:3px}

    </style>

<body onload="getlogo()">
    <div class="hdr">
        <span id="txtlogo"><span>12</span> Titanx</span>
    </div>
    <script type="text/javascript">
        var limg;  
        function getlogo() {  
            limg = new Image(250, 40);  
            limg.onload = gotlogo;  
            limg.src = "logo.png";  
        }  
        function gotlogo() {  
            var logo = document.getElementById("txtlogo");  
            logo.parentNode.replaceChild(limg, logo);  
        }
    </script>
</body>
```

#### 使用图片切片

> 图片切片是将大图片切分成多个较小的图片.可以在链接、脚本或CSS中以这种方式应用图片的一部分,而不是整个图片.但是,这会造成多次往返,会对性能造成严重影响,特别是在图片都相对很小的时候.即使第一个图片比下载整个图片都快,但是往返还是会降低页面的加载速度.
> 但是,对于大图片,切片可以提高感知性能.如果将这些切片分布在多个域名中,网络访问的重叠还是会降低整个页面的加载时间.因此,有时我们要权衡两个指导原则：提高感知性能和减少往返.一般来讲,应该优先考虑感知性能,这是最终真正起作用的.
> 要确定感知性能是否能从切片上获得好处,一个合理的推荐原则是使用与用户差不多的网速,观察页面的下载时间.如果单个图片要花费几秒钟时间,感觉起来比页面其他部分加载得慢,或者让用户觉得需要等待某些东西,那就应该考虑切片.不用对那些加载起来还要不了一秒钟的图片切片,因为用户不会注意到有什么差别,也不要对在折叠区域下面用户看不到的图片进行切片.

#### 指定图片大小属性

> 应该在`<img>`标签的height和widht属性中指定图片的真实大小或比真实大小还大.如果希望图片以更小的样子显示,那最好就在服务器端改变图片大小,避免下载额外的数据.除此之外,像Photoshop等应用程序会使用算法改变图片的大小,其结果一般都会比浏览器随机做的好看.
> 通过指定比实际尺寸更大的大小来拉伸图片通常没什么用处,这需要浏览器做额外的工作,应该避免.如果需要一些额外的填充,可以尝试使用更大的边框或基于CSS的背景颜色.

#### 设置网站图标文件

> 当浏览器下载完网站的第一个页面后,它会请求一个名为favicon.ico的文件.如果提供了该文件,浏览器会将其显示在地址栏,靠在网站URL的左边.应该确保网站提供了该文件.如果没有提供,浏览器将频繁地重新请求,这会导致多次往返和”文件没找到“（file not found）的错误,可以通过在浏览器第一次请求时返回该文件以避免该问题.该文件必须是16像素×16像素,ICO格式（与png、GIF或PNG不同）.

### 减少上传

- 当在页面上请求对象时,减少客户端上传数据的量
- 使用Cookie的path属性,不要讲Cookie与静态文件关联,并使用短的编码的名称和值降低Cookie消耗的带宽和时间

### 预缓存

> 假如可以预先估计到用户最有可能访问的下一个页面,那就可以使用JavaScript预先缓存该页面使用的对象.当用户跳到该页面时,就会加载得很快,因为一些它使用的一些对象已经缓存在客户端了.用户不会注意到缓存,因为它发生在当前页面完成加载之后,网络空闲时.
> 在这里Fiddler会非常有用,它可以用来帮助你找到在当前页面之后的目标页面最常用的对象.
